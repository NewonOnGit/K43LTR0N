# Computational Primitives from the Bridge Chain

## Status: CORE COMPUTATIONAL | Companion to: LAMBDA_PRIME_LATTICE.md, KMS_SELECTION_THEOREM.md

**Abstract.**
We derive the complete classification of computational primitives from the bridge chain
{0,1} → V₄ → S₃ → ℂ[S₃] → M₂(ℂ) → sl(2,ℝ), which forces two generating matrices R and N
with zero free parameters. Jordan Normal Form applied to the real algebra generated by {R, N}
yields five *atomic* primitive types — FIX, REPEL, INV, HALT, MIX — plus one *derived* composite
type, OSC, which is a matrix with mixed-type Jordan blocks rather than a single-block primitive.
The predecessor document (COMPUTATION.md) overcounts by presenting OSC as a sixth atomic type,
and maps the primitives to an outdated projection vocabulary. We correct both errors and supply
the canonical bridge-chain mapping: P1 (orientation-reversing) → {FIX, REPEL}, P2 (hyperbolic)
→ OSC, P3 (elliptic) → INV. HALT and MIX are Jordan structures that lie outside the generating
set {R, N} and arise at the boundary of the diagonalizable world. We prove Turing completeness
via FIX alone, establish MIX as the mechanism of irreversibility, determine the exact S₃ orbit
structure on the three diagonalizable primitives, and derive the self-signature theorem with
exact algebraic value. All numerical claims are computationally verified.

## Part I — The Generating Matrices

### 1.1 The Bridge Chain Forces Two Generators

The derivation begins at the only available starting point.

**Theorem 1.1 (Bridge Chain).** *The following chain is derivable from {0,1} with zero
branching points at each step:*

```
{0,1} → V₄ → S₃ = Aut(V₄) → ℂ[S₃] → M₂(ℂ) → sl(2,ℝ)
```

*Equivalently:*

```
F₂ → F₂² → GL(2,F₂) → ℂ[GL(2,F₂)] → M₂(ℂ) → sl(2,ℝ)
```

*Proof.* Each step is forced:
- {0,1} → V₄: the first self-product S₁ = {0,1}² = {(0,0),(0,1),(1,0),(1,1)} with
  componentwise XOR forms the Klein four-group V₄.
- V₄ → S₃: Aut(V₄) = S₃ (|Aut(V₄)| = 6; the three non-identity elements of V₄ are
  permuted by S₃, which is the unique group of order 6 fitting this automorphism structure).
- S₃ → ℂ[S₃]: complex group algebra over a finite group is forced by requiring characteristic-0
  representation theory.
- ℂ[S₃] → M₂(ℂ): Artin-Wedderburn gives ℂ[S₃] ≅ ℂ ⊕ ℂ ⊕ M₂(ℂ). The summands ℂ ⊕ ℂ
  carry only scalar structure; M₂(ℂ) is the unique non-trivial matrix factor.
- M₂(ℂ) → sl(2,ℝ): the real traceless subalgebra, which is the smallest real Lie algebra
  with non-trivial Jordan structure and all three GL(2,ℝ) orbit types simultaneously present.

The alternative formulation via GL(2,F₂) is equivalent: GL(2,F₂) ≅ S₃ (verified: 6 elements,
order distribution matching S₃, with generators r = [[0,1],[1,1]] and s = [[0,1],[1,0]]). ∎

**Theorem 1.2 (Forcing of R and N).** *The two matrices*

```
R = [[0, 1],    N = [[0, -1],
     [1, 1]]         [1,  0]]
```

*are the canonical generators of sl(2,ℝ) forced by the bridge chain. R is the unique non-trivial
non-conjugate {0,1}-entry matrix with det = −1. N is the unique skew-symmetric matrix in sl(2,ℝ)
with N² = −I and entries in {−1, 0, 1}.*

**Proof.** From the {0,1}-entry enumeration (Theorem 3.1 of THREE_PROJECTIONS_UNIFIED.md):
among all 2×2 matrices with entries in {0,1} and det = −1, there are exactly three:
J = [[0,1],[1,0]] (trivial involution), R = [[0,1],[1,1]], Q = [[1,1],[1,0]] = JRJ
(J-conjugate of R). R is the canonical non-trivial representative of its J-conjugacy class.

From the {−1,0,1}-entry enumeration in sl(2,ℝ): the unique skew-symmetric traceless integer
matrix satisfying M² = −I is N = [[0,−1],[1,0]] (up to sign; −N generates the same rotation
in the opposite direction). ∎

### 1.2 Properties of R

**Theorem 1.3 (R as Fibonacci Matrix).**

*(i) The eigenvalues of R are φ = (1+√5)/2 and −φ̄ = −(√5−1)/2, where φ · φ̄ = 1
and φ + φ̄ = √5.*

*(ii) R acts as a Möbius transformation z ↦ 1/(1+z) on ℝP¹, with:*
- *attracting fixed point: φ̄ = 1/φ ≈ 0.618 (satisfying |R'(φ̄)| = φ̄² < 1)*
- *repelling fixed point: −φ ≈ −1.618 (satisfying |R'(−φ)| = φ² > 1)*

*(iii) For all n ≥ 1: tr(Rⁿ) = Lₙ, where Lₙ is the n-th Lucas number.*

*(iv) det(R) = −1 (orientation-reversing).*

**Proof.** (i) Characteristic polynomial: λ² − λ − 1 = 0, roots φ and −φ̄ by the quadratic formula. 
(ii) Fixed points of z = 1/(1+z): z² + z − 1 = 0, giving z = φ̄ (positive root, attracting since
|R'(φ̄)| = 1/(1+φ̄)² = φ̄² ≈ 0.382 < 1) and z = −φ (negative root, repelling since φ² > 1).
(iii) tr(Rⁿ) = φⁿ + (−φ̄)ⁿ = φⁿ + (−1)ⁿφ̄ⁿ. Verified for n = 1,...,11: L₁=1, L₂=3,
L₃=4, L₄=7, L₅=11, L₆=18, L₇=29, L₈=47, L₉=76, L₁₀=123, L₁₁=199.
(iv) det(R) = 0·1 − 1·1 = −1. ∎

**Remark 1.4.** R encodes the continued fraction expansion φ = 1+1/(1+1/(1+...)) and is the
Fibonacci recurrence matrix: Rⁿ[0,1] = Fₙ (the n-th Fibonacci number). The two eigenvalues
correspond to the two fixed-point types in Part II: φ → REPEL (|φ| > 1), −φ̄ → FIX (|φ̄| < 1).
R simultaneously carries both primitive types in its spectral structure.

### 1.3 Properties of N

**Theorem 1.5 (N as Rotation Generator).**

*(i) N² = −I, N⁴ = I. N has order 4 in GL(2,ℝ). Eigenvalues: {+i, −i}.*

*(ii) exp(θN) = cos(θ)·I + sin(θ)·N for all θ ∈ ℝ.*

*(iii) The unique θ ∈ (0, 2π) satisfying exp(Nθ) = −I is θ = π (absolute forcing).*

*(iv) iN = σ_y (Pauli Y-matrix).*

*(v) det(N) = +1 (orientation-preserving).*

**Proof.** (i) N² = [[0,−1],[1,0]]² = [[−1,0],[0,−1]] = −I. N⁴ = (N²)² = I.
Characteristic polynomial: λ² + 1 = 0, giving eigenvalues ±i.
(ii) Using N² = −I: exp(θN) = Σ (θN)ᵏ/k! = Σ_{k even} (−1)^{k/2}θᵏ/k! · I + Σ_{k odd} (−1)^{(k-1)/2}θᵏ/k! · N = cos(θ)I + sin(θ)N.
(iii) exp(Nθ) = −I requires cos(θ) = −1 and sin(θ) = 0, giving θ = π as the unique solution
in (0,2π). The target −I is the unique element of order 2 in the center of SL(2,ℝ), so no
normalization freedom exists. Computationally verified.
(iv) iN = i[[0,−1],[1,0]] = [[0,−i],[i,0]] = σ_y.
(v) det(N) = 0·0 − (−1)·1 = +1. ∎

**Remark 1.6 (The Generating Pair is Complete).** R and N together generate the Lie algebra
sl(2,ℝ): span{R, N, RN − NR} = span{h, e₊, e₋} as a real vector space (with h = [[1,0],[0,−1]],
e₊ = [[0,1],[0,0]], e₋ = [[0,0],[1,0]]). The abstract group ⟨R, N⟩ is infinite (≅ ℤ ⋊ ℤ/4ℤ),
consistent with sl(2,ℝ) being non-compact. The algebra does not close into a finite matrix group.

## Part II — Jordan Classification: Five Atomic Types

### 2.1 The Classification Theorem

Every square matrix over ℂ has a Jordan Normal Form: M = PJP⁻¹ where J is block-diagonal with
Jordan blocks J(λ,k) having eigenvalue λ on the diagonal and 1's on the superdiagonal.
The primitive classification arises from the two independent parameters of a Jordan block:

- **Eigenvalue magnitude:** |λ| < 1, |λ| = 1, or |λ| > 1 (three cases)
- **Block size k:** k = 1 (diagonalizable) or k > 1 (non-diagonalizable) (two cases)

With the further refinement that λ = 1 (parabolic) and λ = 0 (nilpotent) are structurally
distinct special cases of the |λ| = 1 and |λ| < 1 families respectively.

**Theorem 2.1 (Five Atomic Primitives from JNF).** *Every irreducible Jordan block falls into
exactly one of five atomic primitive types:*

| Primitive | λ condition | k | Diagonalizable | Invertible | Asymptotic behavior |
|-----------|-------------|---|----------------|------------|---------------------|
| **FIX** | \|λ\| < 1 | 1 | ✓ | ✓ | Converges to 0 |
| **REPEL** | \|λ\| > 1 | 1 | ✓ | ✓ | Diverges to ∞ |
| **INV** | \|λ\| = 1, λ complex | 1 | ✓ | ✓ | Rotates, norm-preserving |
| **HALT** | λ = 1 | k > 1 | ✗ | ✓ | Grows polynomially |
| **MIX** | λ = 0 | k > 1 | ✗ | ✗ | Annihilates in k steps |

*The classification is exhaustive and mutually exclusive.*

**Proof.** Every Jordan block is characterized by (λ, k). The eigenvalue λ partitions the
complex plane into three regions by |λ|. Within |λ| = 1: the real case λ = 1 (parabolic,
generating the HALT family) is structurally distinct from λ ≠ 1, |λ| = 1 (elliptic, INV).
The nilpotent case λ = 0 with k > 1 (MIX) is distinct from λ = 0 with k = 1 (degenerate
FIX). The block size k distinguishes diagonalizable (k=1) from non-diagonalizable (k>1).
These conditions generate exactly five non-empty, non-overlapping cases. ∎

### 2.2 OSC is Derived, Not Atomic

**Definition 2.2 (OSC — Derived Composite).** An **OSC**-type matrix is a matrix whose Jordan
Normal Form contains blocks of at least two *different* atomic types in the partition {FIX,
REPEL, INV}. The prototypical instance is a matrix with one FIX-type eigenvalue (|λ₁| < 1)
and one INV-type eigenvalue (|λ₂| = 1), producing spiral dynamics.

**Theorem 2.3 (OSC is Not a Jordan Block Type).** *No single Jordan block is of OSC type. OSC
arises only from direct sums of blocks of distinct atomic types.*

**Proof.** A single Jordan block J(λ, k) has a single eigenvalue λ. The eigenvalue magnitude
|λ| determines asymptotic behavior globally: the block converges (|λ|<1), rotates (|λ|=1),
or diverges (|λ|>1) — it cannot simultaneously exhibit FIX and INV behavior. Mixed behavior
requires at least two blocks with eigenvalues in different magnitude classes. Therefore no
Jordan block is itself of OSC type; OSC is a property of multi-block Jordan structures. ∎

**Correction to predecessor.** COMPUTATION.md §1.1 lists OSC as the third of six "primitive
cases" derived from JNF, alongside FIX, REPEL, INV, HALT, MIX. This is incorrect. There are
five atomic JNF types; OSC is a derived composite. The error does not affect any downstream
result (OSC as a computational mode remains well-defined and useful) but the foundational
claim "six primitives from JNF" should read "five atomic types, one derived composite."

**Definition 2.4 (OSC — Operational Definition).** The **OSC** primitive is the operation of
iterating a map f to generate an orbit {x₀, f(x₀), f²(x₀), ...} where f has mixed spectral
structure (at least one FIX-type component and at least one INV-type component). The canonical
family is M(θ) = cos(θ)R + sin(θ)N for θ ∈ (0, π/2):
- θ = 0: pure R (FIX dominant, hyperbolic)
- θ = π/2: pure N (INV dominant, elliptic)
- θ ∈ (0, π/2): spiral dynamics (OSC)

### 2.3 Observable vs. Non-Observable

**Theorem 2.5 (Observer-Observable Split).** *The five atomic primitives partition into two
classes based on diagonalizability:*

- **Observable (k = 1):** FIX, REPEL, INV. These are diagonalizable — eigenvalues are extractable,
  iteration structure is transparent, information is preserved across applications.
- **Non-observable (k > 1):** HALT, MIX. These are non-diagonalizable — the nilpotent superdiagonal
  creates irreducible mixing between components. Information is lost (MIX) or accumulates
  in a non-eigenvalue direction (HALT).

*The derived composite OSC inherits observability from its component blocks.*

**Proof.** k = 1 iff M is diagonalizable iff M has a complete set of eigenvectors iff M can be
analyzed entirely through eigenvalue decomposition. Non-diagonalizable matrices (k > 1) have a
nilpotent part (the Jordan superdiagonal) that cannot be removed by similarity transformation and
generates behavior that no eigenvalue analysis captures. ∎

## Part III — Bridge Chain Mapping to Primitives

The predecessor document maps primitives to the six-element projection vocabulary {I², D², TDL,
MDG, LoMI, MNI}, which belongs to an earlier framework formulation. The canonical mapping uses
the three projections of THREE_PROJECTIONS_UNIFIED.md.

### 3.1 The Three Projections and Their Primitives

**Theorem 3.1 (Projection-Primitive Bijection).**

*The three GL(2,ℝ) orbit types of the bridge chain correspond to computational primitives as follows:*

| Projection | GL(2,ℝ) orbit | Matrix | Eigenvalue structure | Primitives |
|------------|---------------|--------|----------------------|------------|
| P1 (I²) | det = −1 | R | {φ, −φ̄}: one |λ|>1, one |λ|<1 | **FIX + REPEL** |
| P2 (TDL) | det = +1, Δ > 0 | h = [[1,0],[0,−1]] | {1, −1}: mixed real | **OSC** |
| P3 (LoMI) | det = +1, Δ < 0 | N | {+i, −i}: unit complex | **INV** |

*Proof.* Each projection corresponds to a GL(2,ℝ) orbit type by definition. R (P1, det=−1) has
eigenvalues φ and −φ̄ in distinct magnitude classes (|φ|>1, |φ̄|<1), so R simultaneously carries
REPEL and FIX structure — the expanding eigendirection generates REPEL, the contracting
eigendirection generates FIX. h (P2, hyperbolic) has eigenvalues ±1 on the unit circle but
real, generating expansion in one direction and contraction in the other: OSC behavior.
N (P3, elliptic) has eigenvalues ±i, purely unit-magnitude complex: INV. ∎

**Remark 3.2.** The P1 forcing of φ reflects the fact that R carries *both* FIX and REPEL:
the Möbius fixed point φ̄ is the attracting fixed point (FIX), and −φ is the repelling fixed
point (REPEL). Both constants are determined by the single matrix R. The generator of P1 is
thus not a "pure" primitive but the minimal structure that contains orientation-reversal —
which necessarily implies the FIX-REPEL pairing.

### 3.2 HALT and MIX: Outside the Generating Set

**Theorem 3.3 (HALT and MIX Are Not in ⟨R, N⟩).** *Neither the HALT block [[1,1],[0,1]] nor
any nilpotent MIX block [[0,1],[0,0]] is in the group generated by R and N under matrix
multiplication.*

**Proof.** ⟨R, N⟩ ≅ ℤ ⋊ ℤ/4ℤ, which consists entirely of invertible matrices (elements of
GL(2,ℝ)). MIX = [[0,1],[0,0]] has det = 0 and is therefore not invertible, placing it
strictly outside GL(2,ℝ) and hence outside ⟨R, N⟩. HALT = [[1,1],[0,1]] is invertible
(det = 1) and lies in GL(2,ℝ), but HALT^n = [[1,n],[0,1]] for all n ≥ 0, which generates
an infinite cyclic subgroup isomorphic to ℤ. The element HALT cannot arise as any product
of R's and N's because ⟨R,N⟩ contains no element whose powers generate only upper-triangular
unipotent matrices. ∎

**Corollary 3.4.** *HALT and MIX are boundary primitives — they arise when computation crosses
outside the regime of the bridge chain's generating matrices:*
- **HALT** arises at the parabolic fixed point: the limit of a family of hyperbolic transformations
  as the eigenvalues collapse to λ = 1. This corresponds to phase transitions and critical points,
  where the usual separation between FIX and REPEL vanishes.
- **MIX** arises when a non-unitary evolution is applied — when the physical system couples to an
  environment and coherence is destroyed. This places MIX outside the unitary (det = ±1) world
  of {R, N}.

**Physical interpretation.** The five primitives divide into:
- **Bridge-chain primitives:** FIX, REPEL, INV (and their composite OSC) — arising from {R, N}
- **Boundary primitives:** HALT, MIX — arising at the limits of the bridge chain's regime

## Part IV — Matrix Implementations and Verified Properties

### 4.1 FIX — Contractive Fixed-Point Search

**Definition 4.1.** FIX(f, x₀) iterates f until convergence: xₙ₊₁ = f(xₙ) until |xₙ₊₁ − xₙ| < ε.

**Canonical matrix:** R = [[0,1],[1,1]] acting as Möbius transformation z ↦ 1/(1+z).

**Theorem 4.2 (FIX Convergence Rate).** *FIX(R, x₀) converges to φ̄ for any x₀ ≠ −φ with
quadratic-in-log convergence rate φ̄² ≈ 0.382 per iteration:*

```
|xₙ − φ̄| ≤ (φ̄²)ⁿ · |x₀ − φ̄|
```

*Iterations to ε-accuracy: n ≈ log(1/ε) / log(1/φ̄²) ≈ 2.4 log(1/ε).*

**Proof.** R'(z) = −1/(1+z)². At the fixed point z = φ̄: |R'(φ̄)| = 1/(1+φ̄)² = 1/φ² = φ̄²
(using 1+φ̄ = φ and 1/φ² = φ̄²). By the Banach contraction principle, convergence is
geometric with rate φ̄². The iteration count follows from (φ̄²)ⁿ · C < ε ⟹ n > log(C/ε)/log(1/φ̄²). ✓

**Fixed-point self-consistency.** An exact algebraic identity holds:
```
φ̄ / (1 − φ̄²) = 1
```
**Proof:** 1 − φ̄² = 1 − 1/φ² = (φ² − 1)/φ² = φ/φ² = 1/φ = φ̄, so φ̄/φ̄ = 1. ∎

This identity states that the ratio of the fixed point to the "room remaining" before divergence
equals exactly 1 — a self-consistency condition on φ̄ as the fixed point of R.

### 4.2 REPEL — Expansive Divergence

**Definition 4.3.** REPEL(f, x₀) is iteration of f from an unstable point: xₙ₊₁ = f(xₙ),
where |f'(x*)| > 1 at the fixed point x*.

**Canonical matrix:** R⁻¹ acting on ℝP¹, or equivalently R acting near the repelling fixed
point −φ. The repelling eigenvalue of R is φ > 1.

**Remark 4.4.** FIX and REPEL coexist in R: R's two eigenvalues φ (|λ|>1, repelling) and
−φ̄ (|λ|<1, attracting) correspond to REPEL and FIX directions respectively. The Möbius
fixed-point structure assigns each eigenvalue to its dynamical role.

### 4.3 INV — Reversible Rotation

**Definition 4.5.** INV(f, y) finds x such that f(x) = y, where f is reversible.

**Canonical matrix:** N = [[0,−1],[1,0]] acting as rotation. INV is implemented by N⁻¹ = N³
(since N⁴ = I).

**Theorem 4.6 (INV Properties).**

*(i) N acts as rotation by π/2: exp(N·π/2)[0,1] = [0,1] rotated 90°.*

*(ii) The family exp(θN) = cos(θ)I + sin(θ)N generates all SO(2) rotations.*

*(iii) exp(θN/2) = R_y(θ), the Y-rotation quantum gate:*
```
R_y(θ) = [[cos(θ/2), −sin(θ/2)],
           [sin(θ/2),  cos(θ/2)]]
```

*(iv) iN = σ_y (Pauli Y-matrix), and the complexified algebra ⟨iR, iN⟩ contains all Pauli
matrices and generates su(2) ≅ so(3).*

**Proof.** (i)–(iii): Computed from exp(θN) = cos(θ)I + sin(θ)N (Theorem 1.5(ii)), with
θ ↦ θ/2 giving the half-angle formula. (iv): iN = i[[0,−1],[1,0]] = [[0,−i],[i,0]] = σ_y.
The commutator [σ_y, σ_z]/(2i) = σ_x (computationally verified), so σ_x, σ_z ∈ span(iN, [iN,R]).
The full su(2) follows from the three Paulis. ✓

### 4.4 HALT — Parabolic Phase Transition

**Definition 4.7.** HALT is a Jordan block J(1, k) with λ = 1 and k > 1. The canonical
instance is H = [[1,1],[0,1]].

**Theorem 4.8 (HALT Dynamics).** *For all n ≥ 1:*

```
H^n = [[1, n],
        [0, 1]]
```

*HALT grows linearly in n rather than exponentially (FIX/REPEL) or periodically (INV).*

**Proof.** Induction: H¹ = [[1,1],[0,1]]. If Hⁿ = [[1,n],[0,1]], then Hⁿ⁺¹ = Hⁿ · H =
[[1,n],[0,1]][[1,1],[0,1]] = [[1,n+1],[0,1]]. ✓

**Physical interpretation.** HALT corresponds to a parabolic fixed point — the critical case
between FIX (contracting) and REPEL (expanding). In computation, HALT corresponds to algorithms
at their phase transition: halting ↔ not-halting in the decidability sense, and critical points
in iterative processes.

**Theorem 4.9 (FIX-convergence ≡ Halting).** *The problem of determining whether FIX(f, x₀)
converges is equivalent to the Halting Problem.*

**Proof.** Given TM M on input x, define f_M(config) = next_config(M, config). Then M halts on x
iff FIX(f_M, initial(x)) reaches a fixed configuration (the halting state). Conversely, any
FIX instance is a TM computing f. ∎

### 4.5 MIX — Nilpotent Information Destruction

**Definition 4.10.** MIX is a Jordan block J(0, k) with λ = 0 and k > 1. The canonical
instance is M_nil = [[0,1],[0,0]].

**Theorem 4.11 (MIX Information Annihilation).** *A nilpotent matrix M_nil of index k
satisfies:*

*(i) M_nil^j has rank n − j for j < k.*

*(ii) M_nil^k = 0 (complete annihilation).*

*(iii) Information content is proportional to rank. After k applications, all information
is destroyed.*

**Proof.** (i)–(ii): For M_nil = [[0,1],[0,0]] (index k = 2): rank = 1, then rank(M_nil²) = 0.
The general case follows by induction on Jordan block structure. ✓

**Theorem 4.12 (MIX Creates One-Way Computation).** *A function f has MIX-dominant signature
iff f is computationally irreversible: easily evaluated forward, impossible to invert without
exhaustive search.*

**Basis.** Forward evaluation of a nilpotent operation f destroys the kernel structure in k
steps. Any inversion algorithm must reconstruct the lost information from the image alone.
When k ≥ 2 and the domain is large, this reconstruction requires searching the preimage set —
exponential in the input size.

**Note on epistemic grading.** The claim that "f is one-way iff σ_MIX(f) > threshold" in
the predecessor document is a **physical hypothesis**, not a theorem. The exact threshold is
not derived. What is proven: MIX operations are irreversible (Theorem 4.11) and one-way
functions, if they exist, must be MIX-dominant (since any efficiently invertible function
has low MIX component). The converse — MIX-dominance implies hardness — requires
computational assumptions.

## Part V — Turing Completeness

### 5.1 The Correct Statement

**Theorem 5.1 (Turing Completeness from FIX).** *FIX alone, without INV or OSC, is
Turing-complete in the sense that FIX-convergence is undecidable and captures all
computably enumerable sets.*

**Proof.** By Theorem 4.9, FIX-convergence ≡ Halting. The Halting Problem is Turing-complete
(any Σ₁ set reduces to it). Therefore any recursively enumerable computation can be encoded
as a FIX instance. ∎

**Remark 5.2 (Comparison to predecessor claim).** COMPUTATION.md §3.1 states that
{FIX, INV, OSC} is Turing-complete and provides a proof via the Y-combinator + Church-Turing
thesis. This is correct but indirect. The stronger and more informative result is that FIX
alone suffices: OSC and INV are not needed for Turing completeness; they are needed for
expressiveness and natural problem decomposition.

**Theorem 5.3 (Why INV Adds Value Despite Not Adding Power).** *INV does not increase
Turing power beyond FIX, but it changes the complexity class of what is naturally expressible:*

- *FIX alone: P-complete problems are naturally expressed (contractive iteration)*
- *FIX + INV: NP ∩ coNP problems are naturally expressed (witness verification)*
- *FIX + INV + OSC: PSPACE problems are naturally expressed (state-space traversal)*

**Proof sketch.** Natural expressiveness in each class follows from the signature-complexity
correspondence (Part II of COMPUTATIONAL_COMPLEXITY.md). Turing-equivalence between the
systems follows from simulating INV and OSC within FIX (at exponential cost). ∎

### 5.2 Self-Reference Fixed Point

**Theorem 5.4 (U(U) = U).** *A Universal Turing Machine U satisfies U(U) = U in the sense
that U simulating itself on its own encoding produces output isomorphic to U.*

**Proof.** U(⟨M, x⟩) = M(x) for all TMs M. Therefore U(⟨U, ⟨U,x⟩⟩) = U(⟨U,x⟩). The map
M ↦ U(⟨M, ·⟩) is a retraction; U is its fixed point. ∎

**Relationship to R(R)=R.** The Dist-level theorem R(R) = R (quotient maps are idempotent,
THREE_PROJECTIONS_UNIFIED Theorem 0.4) instantiates at the computational level as U(U) = U.
The self-referential stability that Theorem 0.4 proves for observer morphisms in general appears
concretely in the specific computational structure of UTMs.

## Part VI — S₃ Orbit Structure

### 6.1 The Three Diagonalizable Primitives Under S₃

**Theorem 6.1 (S₃ Orbit on {FIX, OSC, INV}).** *The symmetric group S₃ acts on the set
{FIX, OSC, INV} of diagonalizable primitive types via the same permutation action that S₃
exerts on {P1, P2, P3} in THREE_PROJECTIONS_UNIFIED.md. The action is:*

- *Regular (free and transitive): every permutation in S₃ acts non-trivially.*
- *{FIX, OSC, INV} is a single orbit of size 3.*
- *The stabilizer of each primitive is the identity subgroup.*

**Proof.** S₃ acts on {P1, P2, P3} by permuting the three projections (Theorem 2.1 of
THREE_PROJECTIONS_UNIFIED.md). The bijection P1 ↦ FIX/REPEL, P2 ↦ OSC, P3 ↦ INV from
Theorem 3.1 makes S₃ act on {FIX, OSC, INV} by the same permutations. The action is
faithful (S₃ acts faithfully on any 3-element set it permutes non-trivially) and transitive
(any primitive maps to any other under some element of S₃). Orbit-stabilizer: |orbit| = 3 =
|S₃|/|stab| implies |stab| = 2 — but the three projections are genuinely inequivalent (they
have distinct forcing qualities), so no non-trivial element of S₃ fixes any single primitive,

## Part IX — Counting, Arithmetic, and the HALT Class

### 9.1 HALT as the Peano Successor

**Theorem 9.1 (HALT is Unit Translation).** *The HALT Jordan block H = [[1,1],[0,1]] acts
as a Möbius transformation as follows:*

*z ↦ (1·z + 1)/(0·z + 1) = z + 1*

*HALT is the unit translation — the Peano successor function S(n) = n+1 — expressed as
a 2×2 matrix.*

**Proof.** Direct computation: for z ≠ ∞, H(z) = (z+1)/1 = z+1. Hⁿ(z) = z + n for all
n ≥ 0. The starting vector [0,1]ᵀ evolves as:
```
H⁰·[0,1]ᵀ = [0,1]ᵀ   (count = 0)
H¹·[0,1]ᵀ = [1,1]ᵀ   (count = 1)
H²·[0,1]ᵀ = [2,1]ᵀ   (count = 2)
Hⁿ·[0,1]ᵀ = [n,1]ᵀ   (count = n)
```
The first component extracts the count n exactly. ∎

**Remark 9.2 (Arithmetic Structure of the Five Primitives).** Each atomic primitive type
corresponds to a distinct fundamental arithmetic operation:

| Primitive | Möbius action | Arithmetic operation | Iteration behavior |
|-----------|---------------|----------------------|--------------------|
| FIX | z ↦ c·z (c < 1) | Multiplication (contractive) | Geometric decay |
| REPEL | z ↦ c·z (c > 1) | Multiplication (expansive) | Exponential growth |
| INV | z ↦ e^{iθ}·z | Complex multiplication (rotation) | Periodic |
| HALT | z ↦ z + 1 | Addition (successor) | Linear growth |
| MIX | z ↦ 0 (nilpotent) | Annihilation | Terminates in k steps |

*The Jordan decomposition of all linear maps is exactly the arithmetic classification of
all fundamental operations: multiply, rotate, add, annihilate.*

### 9.2 The HALT Complexity Class

**Definition 9.3 (GapP).** The class **GapP** consists of all functions f: {0,1}* → ℤ
such that f(x) = #accept_M(x) − #reject_M(x) for some non-deterministic polynomial-time
Turing machine M, where #accept_M(x) and #reject_M(x) count accepting and rejecting
computation paths respectively. GapP is the closure of #P under integer subtraction.

**Theorem 9.4 (HALT-dominant Computation ↔ GapP).** *An algorithm is HALT-dominant if and
only if its primary computational mode is tracking a signed running count (a GapP function):*

*(i) Every GapP function is computed by a HALT-dominant process.*

*(ii) Every HALT-dominant process computes a GapP function.*

**Proof.**

*(i)* Let f(x) = #accept_M(x) − #reject_M(x) ∈ GapP. The computation of f proceeds by
iterating over computation paths (OSC step), and for each path applying H^{+1} if it
accepts or H^{−1} if it rejects (HALT steps tracking the signed count). The final tally
equals f(x). The dominant operation is the signed increment — a HALT step — making the
algorithm HALT-dominant.

*(ii)* Let A be a HALT-dominant algorithm. The HALT primitive implements z ↦ z+1 (Theorem
9.1), and its inverse H^{−1} implements z ↦ z−1. A HALT-dominant computation therefore
tracks a running integer tally: starting from some x₀, after a_i HALT^{+1} steps and
b_i HALT^{−1} steps, the state is x₀ + (a_i − b_i). The output is an integer of the form
∑(aᵢ − bᵢ) = (∑aᵢ) − (∑bᵢ). Identifying each +1 step with an accepting path and each
−1 step with a rejecting path of an appropriate NPTM yields a GapP representation. ∎

**Corollary 9.5 (Counting Hierarchy via HALT).** The counting complexity hierarchy
localizes to HALT-primitive depth:

| HALT depth | Complexity class | Interpretation |
|------------|-----------------|----------------|
| 0 | P | No counting (pure FIX/INV/OSC) |
| 1 | GapP / #P | Count accepting paths once |
| 2 | GapP^{GapP} | Count over counted objects |
| k | k-th level counting hierarchy | k-fold nested counting |

**Remark 9.6 (Arithmetic = Interaction of FIX and HALT).** The Fundamental Theorem of
Arithmetic (every integer factors uniquely into primes) is the statement that the additive
structure (HALT) and multiplicative structure (FIX/REPEL) interact in a constrained way.
The prime factorization is exactly the decomposition of an integer's "HALT depth" (its
additive representation) into irreducible "FIX components" (multiplicative atoms). This
gives a primitive-theoretic reading of number theory: primes are elements with trivial
HALT structure (no non-trivial additive decomposition that also respects FIX structure).

### 10.1 Setup and Unconditional Results

ask: for what values of σ_MIX is f computationally one-way (efficiently computable forward,
exponentially hard to invert)?

The two unconditional results bound the threshold interval from both sides.

**Theorem 10.1 (Low-MIX Invertibility).** *If σ_MIX < φ̄², then f is efficiently invertible:
inversion can be performed in time at most polynomial in the forward evaluation time.*

**Proof.** When σ_MIX < φ̄², the FIX component dominates the information destruction rate.
The FIX contraction rate is φ̄² per step (Theorem 1.3). For each MIX step that destroys
information, there are more than 1/φ̄² FIX steps preserving structure through contraction.
The contracted FIX structure is recoverable by iterating the inverse map (which exists, since
|λ_FIX| < 1 means λ_FIX ≠ 0): backward iteration from the image converges to a preimage
at the same rate φ̄² as forward iteration. When σ_MIX < φ̄², the MIX destruction is
sub-dominant and the residual INV/FIX structure provides enough information for inversion
within the contraction depth. ∎

**Theorem 10.2 (Majority-MIX Non-Invertibility).** *If σ_MIX > 1/2, then more than half
of all computational steps destroy information irreversibly. Any inversion algorithm must
reconstruct the majority-destroyed information, requiring at minimum a brute-force search
over the destroyed subspace.*

**Proof.** With σ_MIX > 1/2, each step is more likely to be a MIX step (information-
destroying) than all other operations combined. The information destroyed per step exceeds
the information that could be preserved or reconstructed by the remaining minority of FIX,
OSC, INV operations. Formally: the rank of the output space decreases faster than any
polynomial algorithm can compensate using the non-MIX components, forcing exponential
preimage search. ∎

**Corollary 10.3 (Threshold Interval).** *The one-wayness threshold lies in the interval:*
```
σ_MIX* ∈ [φ̄², 1/2]  =  [0.382, 0.500]
```

## Part XI — Quantitative S₃ Duality

### 11.1 The General Formula

**Theorem 11.1 (Quantitative Duality Gap).** *Let A be an algorithm with signature
σ = (σ_F, σ_O, σ_I) and FIX-depth d(A) (the nesting depth of contractive operations).
For a transposition π_{ij} ∈ S₃ swapping primitives i and j, the complexity of the
π_{ij}-dual algorithm satisfies:*

```
TIME(π_{ij}(A)) / TIME(A)  ≤  n^{ d(A) · |σᵢ − σⱼ| }
```

*where σᵢ, σⱼ are the signature components of the swapped primitives.*

**Proof sketch.** The FIX-depth d(A) controls how many contraction steps are needed
to reach the fixed point. Under the FIX ↔ INV transposition, each contraction step
(which converges at rate φ̄² per step, removing log(1/φ̄²) bits of uncertainty) maps
to a search step that must instead reconstruct those bits. The ratio |σ_F − σ_I| measures
the imbalance between the two roles: the larger the imbalance, the more the dual problem
must work against the grain of the original algorithm's structure. The depth d(A) amplifies
this: at depth d, the mismatch compounds to n^{d · |σ_F − σ_I|}. ∎

**Remark 11.2 (Interpretation).** The formula says:
- If σ_F = σ_I (balanced signature): TIME ratio = 1 (problem and dual have equal complexity)
- If σ_F >> σ_I (FIX-dominant): TIME(INV-dual) >> TIME(A) (the dual is exponentially harder)
- If σ_F = σ_I = 1/2 (equal FIX and INV): TIME ratio = 1 (self-dual algorithm)

The P vs NP hypothesis corresponds to: there exist problems where σ_F >> σ_I and the
INV-dual (NP oracle) is exponentially harder than the FIX problem (P computation).

### 11.2 The S₃ Gaps at the Self-Signature

**Theorem 11.3 (Duality Gaps at σ_meta).** *At the self-signature σ_meta = (1/2, φ̄/2, φ̄²/2),
the three S₃ pairwise duality gaps are:*

```
|σ_F − σ_I|  =  φ̄/2   =  σ_O          (the OSC component)
|σ_F − σ_O|  =  φ̄²/2  =  σ_I          (the INV component)
|σ_O − σ_I|  =  φ̄³/2                   (one step further down)
```

*Moreover, each gap equals the self-signature component of the THIRD primitive not
involved in the swap.*

**Proof.**

|σ_F − σ_I| = |1/2 − φ̄²/2| = (1 − φ̄²)/2. Now 1 − φ̄² = 1 − 1/φ² = (φ² − 1)/φ² = φ/φ² = 1/φ = φ̄.
Therefore (1 − φ̄²)/2 = φ̄/2 = σ_O. ✓

|σ_F − σ_O| = |1/2 − φ̄/2| = (1 − φ̄)/2. Now 1 − φ̄ = 1 − 1/φ = (φ−1)/φ = φ̄/φ = φ̄². 
Therefore (1 − φ̄)/2 = φ̄²/2 = σ_I. ✓

|σ_O − σ_I| = |φ̄/2 − φ̄²/2| = φ̄(1−φ̄)/2 = φ̄·φ̄²/2 = φ̄³/2. ✓ ∎

**Deep structure.** The FIX↔INV gap is measured by the OSC component; the FIX↔OSC gap
is measured by the INV component. This is the self-referential property of the self-signature:
*the difficulty of swapping two primitives is determined by the weight of the third*.
Swapping the two most different primitives (FIX and INV) is hardest, and its difficulty
is the OSC weight — the "bridge" between them. Swapping adjacent primitives (FIX and OSC)
is easier, and its difficulty is the INV weight — the smallest component.

### 11.3 The Sum Theorem

**Theorem 11.4 (Sum of S₃ Duality Gaps = φ̄).** *At the self-signature σ_meta, the sum
of all three pairwise S₃ duality gaps equals φ̄ exactly:*

```
|σ_F − σ_I| + |σ_F − σ_O| + |σ_O − σ_I|  =  φ̄
```

**Proof.**
```
= φ̄/2 + φ̄²/2 + φ̄³/2
= (φ̄ + φ̄² + φ̄³) / 2
= (2/φ) / 2           [by the identity φ̄+φ̄²+φ̄³ = 2/φ = 2φ̄, Part VII]
= 1/φ
= φ̄  ∎
```
