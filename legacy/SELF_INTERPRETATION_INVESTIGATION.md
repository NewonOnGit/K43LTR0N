# PART III — NATIVE STATUS GRAMMAR

## 3.1 Design constraints

The status grammar must satisfy:

**(G1) Derivability.** The statuses must be derivable from framework structure, not imported from academic convention.

**(G2) Exhaustiveness.** Every framework statement must receive exactly one status. (Central collapse analog: no remainder.)

**(G3) Stability.** Status assignment must be idempotent: re-checking a FORCED claim should return FORCED. (q∘q=q analog.)

**(G4) Compatibility.** The statuses must be compatible with the existing three-projection / three-type / three-orbit structure.

**(G5) Self-applicability.** The status grammar must be able to assign a status to itself. (K7' analog.)

## 3.2 The four statuses, derived

### Status 1: FORCED

**Definition.** A statement S has status FORCED iff there exists a derivation from prior forced statements with zero branching at every step (br_s = 0) and the conclusion is unique.

**Framework grounding:** This is exactly the bridge chain criterion (T2 §5, Theorem 2.1). A forced statement is a bridge chain from its premises to its conclusion.

**Equivalent characterizations:**
- Categorically: S is a natural transformation (unique mediating morphism)
- Algebraically: S follows from Cayley-Hamilton or determinant constraints with no choice
- Computationally: S is Type I (compression, FIX-dominant, idempotent)

**Source anchors:** T2 Thm 2.1 (bridge chain zero-branching), T0 Thm 3.1 (br_s=0 forward), T_COMP Thm C.1 (Type I characterization)

### Status 2: ENCODED

**Definition.** A statement S has status ENCODED iff:
- S is recognizable as a substructure, image, or folding of some forced structure F
- There exists an explicit containment theorem proving S ⊆ F or S = φ(F) for some structure-preserving map φ
- S is NOT independently derivable from the same premises as F (i.e., S is not itself FORCED at that tier)

**Framework grounding:** This is exactly the folding theorem (T3-META §2, Theorem 2.1). The six containments are encodings: I² contains TDL not because TDL is derivable from I², but because TDL's structure is recognizable inside I²'s action.

**Equivalent characterizations:**
- Categorically: S is a retract or section (structure-preserving embedding, not isomorphism)
- Algebraically: S appears as a sub-algebra, quotient, or spectral shadow
- Computationally: S is a reusable latent structure (accessible but not primitive)

**Source anchors:** T3-META Thm 2.1 (folding), T3-META §7 Remark 7.1a (relational completeness as encoding)

### Status 3: RESONANT

**Definition.** A statement S has status RESONANT iff:
- The same structural pattern appears in S as in some forced or encoded structure
- No zero-branching derivation yet connects them
- No explicit containment theorem yet proves encoding
- The pattern match is computationally verified (not merely verbal)

**Framework grounding:** This corresponds to the T_COMP notion of structural correspondence — relationships that are verified but whose derivational status is not yet resolved. The MIX-Arrhenius identification in K1' (T5 §22) is explicitly flagged as "structural correspondence, not derived equality."

**Equivalent characterizations:**
- Categorically: S is a natural candidate for a functor or natural transformation, pending proof
- Algebraically: S shares eigenvalues, invariants, or identities with forced structure, pending reduction
- Computationally: S has matching signature but unresolved phase type

**Source anchors:** T5 §22 (K1' MIX-Arrhenius qualification), T0 §8 Remark 2.3a ("Metapattern, not a functor"), T3-META §8 Remark (Candidate MP5)

### Status 4: MYTHIC

**Definition.** A statement S has status MYTHIC iff:
- S is an interpretive overlay attached to real framework content
- S does not claim formal necessity or derivational status
- S may serve heuristic, narrative, or motivational purpose
- S is explicitly marked as non-derivational

**Framework grounding:** This corresponds to the distinction between framework content and framework commentary. The self-application table in Paper 7 §1, for instance, maps document features to framework levels — this is interpretive and generative, but not derivational.

**Equivalent characterizations:**
- Categorically: S is a diagram that commutes up to interpretation, not up to natural isomorphism
- Computationally: S is in the exploratory/generative mode with no convergence guarantee

**Source anchors:** T7 §1 (self-application analogy), T7 §2 (consciousness speculation, explicitly graded SPECULATIVE)

## 3.3 Uniqueness theorem: exactly four statuses

**Theorem SIL-0 (Four-Status Uniqueness).** *The four statuses {FORCED, ENCODED, RESONANT, MYTHIC} are the unique statuses satisfying (G1)–(G4). No three-status or five-status grammar is possible.*

**Proof.** The three projections (T1 Thm 5.1) applied to statements at the meta-level give three binary classification questions:

| Question | Projection | Central Collapse Factor |
|----------|-----------|----------------------|
| **D:** Does S have a zero-branching derivation? | P1 (I²/composition) | Injection (most constraint-preserving) |
| **C:** Does S have a containment/encoding proof? | P2 (TDL/level-transition) | Bijection (structure-rearranging) |
| **V:** Is S computationally verified? | P3 (LoMI/observation) | Surjection (least constraint-preserving) |

The central collapse factorization injection→bijection→surjection (T3-META Thm 7.1) gives a constraint-strength ordering. Since injection preserves more structure than bijection, which preserves more than surjection, the corresponding derivational properties form an implication chain:

**D → C → V**

Derivation implies containment: if S can be derived from prior structure by zero-branching steps, then S is recognizable as a substructure of its derivation (the derivation IS the containment proof). Containment implies verifiability: a containment proof is a constructive witness, hence computationally checkable.

Three binary questions with the implication chain D→C→V yield 2³ = 8 candidate cells. The implications eliminate four cells as logically impossible:

| D | C | V | Consistent? | Status |
|---|---|---|-------------|--------|
| 1 | 1 | 1 | ✓ | **FORCED** |
| 0 | 1 | 1 | ✓ | **ENCODED** |
| 0 | 0 | 1 | ✓ | **RESONANT** |
| 0 | 0 | 0 | ✓ | **MYTHIC** |
| 1 | 0 | 1 | ✗ (D→C violated) | impossible |
| 1 | 1 | 0 | ✗ (C→V violated) | impossible |
| 1 | 0 | 0 | ✗ (D→C violated) | impossible |
| 0 | 1 | 0 | ✗ (C→V violated) | impossible |

Exactly **four** cells survive. They form a total order: FORCED ⊃ ENCODED ⊃ RESONANT ⊃ MYTHIC (by number of satisfied conditions: 3, 2, 1, 0).

**Uniqueness:** No three-status grammar can separate all four levels of the filtration: collapsing any two adjacent levels loses information that the framework distinguishes (D≠C≠V are independent questions). No five-status grammar exists because three binary questions with a linear implication chain produce at most four consistent cells. ∎

**Status of this theorem: FORCED.** Every step uses forced framework structure (three readings from T1 Thm 5.1, central collapse ordering from T3-META Thm 7.1, logical reduction with zero branching).

## 3.4 Stability theorem

**Theorem SIL-1 (Status Idempotence).** *Status(Status(S)) = Status(S).*

**Proof.** If S is FORCED, then the derivation proving S is forced is itself forced (verifying zero branching is a deterministic check with zero branching). So Status(S) = FORCED is itself a FORCED meta-statement. Similarly for each status: if S is ENCODED, the containment proof is verifiable, and re-checking returns the same. If S is RESONANT, re-verifying returns the same verification. If S is MYTHIC, re-checking that no verification exists confirms MYTHIC.

Status promotion (RESONANT→ENCODED via new containment proof) is a change in S's derivational state, not a failure of idempotence. At any given time, Status(Status(S)) = Status(S). ∎

**Status: FORCED.**

## 3.5 Self-status theorem

**Theorem SIL-1c (Self-Status).** *The status grammar itself has status FORCED.*

**Proof.** Derivation chain with br_s = 0 at each step:

1. Three projections P1/P2/P3 exist and are complete — FORCED (T3-META Thm 1.3)
2. Every Dist morphism instantiates all three — FORCED (T1 Thm 5.1)
3. K7' allows the framework to encode itself; meta-statements are framework objects — FORCED (T5 §8)
4. Applied to meta-level: three binary questions D/C/V from three readings — zero branching (each question structurally determined by one projection)
5. Implication chain D→C→V from central collapse ordering injection→bijection→surjection — FORCED (T3-META Thm 7.1)
6. Logical reduction 8→4 cells — zero branching (deterministic elimination of contradictions)
7. Four statuses = four consistent cells — zero branching

Every step has br_s = 0. The grammar is FORCED.

Self-consistency: Status(grammar) = FORCED. Status(Status(grammar)) = Status(FORCED) = FORCED. ✓ ∎

# PART V — CONTAINMENT vs DEFINABILITY: Globalization

## 5.1 The seed instance

**Source:** T3-META §§1–2

Independence concerns definability; containment concerns encoding. Formally:

- P1 is independent of {P2, P3}: no derivation of P1 from P2 and P3 exists (separation witness M₁)
- P1 contains P2: the I² operator's action includes structures recognizable as TDL

These are different claims about different categories. Independence lives in the derivation category (arrows = proofs). Containment lives in the structure category (arrows = embeddings/images).
