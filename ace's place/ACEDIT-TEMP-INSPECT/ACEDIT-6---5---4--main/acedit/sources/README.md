# ACEDIT Font Sources

Input directory containing the UFO 3 source and PUA allocation table.

## Contents

```
sources/
├── allocation_table.json           1,094-entry PUA codepoint allocation
└── ACEDIT-Regular.ufo/             UFO 3 font source (THE single input)
    ├── metainfo.plist              formatVersion: 3
    ├── fontinfo.plist              All font metrics (derived from phi)
    ├── lib.plist                   Glyph order seed, build config
    ├── layercontents.plist         Single layer: "public.default" -> "glyphs"
    ├── features.fea                OpenType features (deferred to v2.0)
    ├── groups.plist                12 glyph classes for GSUB/GPOS
    └── glyphs/
        ├── contents.plist          Glyph name -> filename map (1,097 entries)
        ├── _notdef.glif            .notdef rectangular frame outline
        ├── space.glif              U+0020, empty, advance 618
        ├── nbspace.glif            U+00A0, empty, advance 618
        └── (1,094 PUA .glif files)
```

## allocation_table.json

Generated by `scripts/generate_allocation.py`. Maps each glyph to its PUA codepoint, block, and addressing indices.

```json
{
  "name": "acedit.apl.m0s0o0d0",
  "codepoint": "U+E000",
  "block": "APL_CORE",
  "machine": 0, "spiral": 0, "operator": 0, "domain": 0
}
```

## UFO 3 Format

The `.ufo` directory is the standard Unified Font Object format (version 3). It is the sole input to fontmake during compilation.

**Important:** `features.fea` references ligature target glyphs that do not yet exist (v2.0). It must be moved aside before compilation to avoid fontmake errors. The build pipeline handles this automatically.

## Regeneration

Glyph sources are regenerated by Stages 1-2 of the build pipeline:

```bash
python3 scripts/generate_allocation.py   # allocation_table.json
python3 scripts/assemble_apl.py          # 972 APL .glif files
python3 scripts/assemble_non_apl.py      # 122 non-APL .glif files
python3 scripts/generate_groups.py       # groups.plist
```
