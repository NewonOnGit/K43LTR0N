OpenType Features Reference

Complete documentation of GSUB and GPOS feature tables defined in features.fea and glyph classes generated by generate_groups.py (modules 10 and 11). All kerning values derive from the phi-based metric system.

Glyph Classes Overview 12 classes

The font defines 12 glyph classes in groups.plist. @PUA_ALL (1,094 glyphs) is the superset. @APL_M0 through @APL_M8 partition the 972 APL_CORE glyphs into 9 machine classes of 108 each. @GOV_GLYPHS and @GEOMETRY_PENT cover the GOVERNANCE and pentagonal GEOMETRY blocks respectively.

Kerning kern

Five class-based kerning rules. All values are multiples of two fundamental ACEDIT spacing units derived from phi: 73 = primary stroke (UPM × φ-4 / 2) and 36 = deficit angle (tight sidebearing). 146 = 2 × 73 = wide sidebearing.

CSS Integration Guide

How to load and use ACEDIT in web projects. The font uses the Private Use Area (U+E000..U+E445) and is designed to work alongside standard Latin text fonts.

@font-face Declaration

@font-face { font-family: 'ACEDIT'; src: url('fonts/ACEDIT-Regular.woff2') format('woff2'), url('fonts/ACEDIT-Regular.otf') format('opentype'); unicode-range: U+E000-F8FF; font-display: swap; }

Profile Activation Classes

/* Default: Theta_standard (no feature activation needed) */ .acedit-text { font-family: 'ACEDIT', monospace; } /* Theta_open: expose all PUA including experimental */ .theta-open { font-feature-settings: "ss07" 1; } /* Theta_restricted: hide non-core (M3-M8 -> .notdef) */ .theta-restricted { font-feature-settings: "ss08" 1; } /* Domain-specific stylistic sets */ .domain-neural { font-feature-settings: "ss01" 1; } .domain-spin-glass { font-feature-settings: "ss02" 1; } .domain-visual { font-feature-settings: "ss03" 1; } .domain-algebra { font-feature-settings: "ss04" 1; } .domain-geometry { font-feature-settings: "ss05" 1; } .domain-ucf { font-feature-settings: "ss06" 1; }

Font Stack Recommendation

/* ACEDIT handles PUA glyphs via unicode-range; the browser falls through to the next font for Latin text. Pair with a monospace for code contexts or a serif for prose. */ /* For code / technical notation */ .acedit-code { font-family: 'ACEDIT', 'JetBrains Mono', monospace; } /* For mixed prose with embedded PUA symbols */ .acedit-prose { font-family: 'ACEDIT', 'Cormorant Garamond', Georgia, serif; }

Example Usage

<!-- Render PUA glyphs in restricted mode with neural domain --> <div class="acedit-code theta-restricted domain-neural"> <!-- U+E000: first APL glyph (machine 0, spiral 0, op 0, domain 0) --> &#xE000;&#xE001;&#xE006; </div> <!-- Ligature composition: phi + infinity = bloom --> <p class="acedit-prose"> The UCF bloom operator (&#xE3CC;&#xE3CD;) represents closure. </p> <!-- Combine profile + domain features --> <span style="font-feature-settings: 'ss07' 1, 'ss05' 1;"> <!-- Theta_open + Geometry domain --> </span>
